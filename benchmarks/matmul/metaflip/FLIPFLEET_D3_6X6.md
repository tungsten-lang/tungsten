# Bounded 6x6 C3 x Z2 experiment

## What is constrained

The worker preserves the exact six-image set generated by

```
rho(u,v,w)   = (v, transpose(w), transpose(u))
sigma(u,v,w) = (reverse(u), reverse(v), reverse(w)).
```

`rho` is the existing C3 matrix-multiplication symmetry. `sigma` reverses both
matrix coordinates in every factor. They commute in this storage convention,
so the implementation calls the constraint C3 x Z2. Earlier campaign notes
also called the six-image search “D3”; the code keeps that short historical
name without claiming the two abstract groups are isomorphic.

The checked-in rank-153 density-2502 and density-2574 schemes are invariant
under both generators. The exact host gate independently checks nonzero and
in-range factors, duplicate freedom, complete tensor reconstruction, C3
closure, and reversal closure before either dispatch or output.

## Equal-compute comparison

The comparison used four 6x6 trials per arm. Every trial received 128 walkers
and 5,000 steps, or 640,000 attempted transitions. The two seeds were the exact
rank-153 density-2502 and density-2574 schemes; each was tested with plus
periods 200 and 50. Thus every arm received 2,560,000 configured walker-steps.

| arm | final rank distribution | final densities | measured kernel time / rate |
|---|---:|---|---:|
| C3 quotient | `{153: 4}` | `2502, 2502, 2502, 2511` | 4.671 s / 548k steps/s |
| C3 x Z2 quotient | `{153: 4}` | `2502, 2502, 2502, 2502` | 7.963 s / 321k steps/s |
| fixed-cube break + generic | `{153: 4}` | `2502, 2502, 2539, 2528` | same 2.56M configured walker-steps |

All emitted candidates passed their engine’s exhaustive host reconstruction;
the C3 x Z2 outputs additionally passed both symmetry gates. The deliberate
break seeds were exact rank 154 with reversal defect 2 or 3, and the generic
arm returned all four to rank 153. No arm found rank 152.

The C3 and C3 x Z2 kernel timers are directly comparable attempted-transition
counts, but their wall rates were measured while an unrelated 7x7 fleet was
also using the GPU, so the exact ratio should be treated as directional. The
six-image kernel delivered about 59% of C3 throughput. It did recover the known
density-2502 representative reliably from the older density-2574 seed, but it
did not improve the current density-2502 leader.

## Scheduling decision

The result does not justify a permanent or default lane. The bounded worker is
kept as a standalone 6x6-only opt-in experiment, not as a continuous role or a
member of the rotating production pool. Deliberate symmetry breaking remains
the faster complementary route into the ordinary graph. A future campaign can
revisit this decision if a different seed or kernel changes the measured
reward/throughput tradeoff.

The runtime ABI is:

```
d3-worker seed output walkers steps dispatches band plus_period
```

Both the bundle glue and executable cap walkers at 4,096, steps at 1,000,000,
dispatches at 64, and band at `CAP-12`. Campaign runtime uses only the
checked-in Tungsten and Metal files; `d3_gpu_worker_gen.py` is an offline
reproducibility tool.
