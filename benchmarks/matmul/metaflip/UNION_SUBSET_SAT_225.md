# Fixed-dictionary rank search for `<2,2,5>`

## Claim boundary

The rigorous interval remains

```text
17 <= R_GF(2)(<2,2,5>) <= 18.
```

The experiments here found no rank-17 scheme. Solver timeouts are not UNSAT
claims, and the finite dictionary is not the universe of rank-one tensors.
The checked lower endpoint is replayed by
`proof_inner2/n225_verify_wang.py`; the five local rank-18 files provide the
upper endpoint and the seed dictionary.

## Deterministic dictionaries

[`flipfleet_225_union_subset_sat.w`](flipfleet_225_union_subset_sat.w) inserts
terms in a fixed order: the five production doors, followed by either the
32 indices recorded by the maximin archive or parents `0..N-1` from
`flipfleet_225_block_gl_parent_lib.w`. A dense exact `(u,v,w)` interner removes
duplicate terms. A separate 400-bit elimination, which does not allocate the
quadratic provenance nullspace, measures the column span.

| dictionary | unique terms | column rank | nullity | serialized-order SHA-256 |
|---|---:|---:|---:|---|
| five doors | 55 | 51 | 4 | `4825c896a4c0c2b1e0d45c0c98de689da456b6a2f54a6e2d6e4ed31815bc7835` |
| five doors + maximin 32 | 625 | 212 | 413 | `94d63c3360f86b94b4d89e3433fb16dd1aa7d8613684e60e12f51e4afc77c1cd` |
| five doors + all 4,096 parents | 3,321 | 212 | 3,109 | `6b953ee7803d479a5d70db923a73abbfaddce6c0eafd7a1a35782d8ef9dab4c5` |

The maximin archive already reaches the full 212-dimensional tensor-column
span of all 4,096 parents. The remaining parents add dictionary coordinates,
not new column-space dimensions, so they can still create new sparse subsets.

## Exact-XOR plus `weight <= 17`

Each dictionary term gets one Boolean variable. For every supported tensor
coefficient the Tungsten emitter writes the exact native CryptoMiniSat XOR;
unsupported zero rows are tautologies and are omitted. A canonical Sinz
sequential counter enforces `sum(x) <= 17`. It does **not** assume that the
answer has exactly seventeen terms. On SAT, Tungsten materializes the chosen
rank-one terms, reconstructs all 400 coefficients, writes the scheme, reparses
it, and reconstructs it again before reporting a hit.

[`audit_225_union_subset_sat.py`](audit_225_union_subset_sat.py) is an
independent implementation of the term expansion, target tensor, GF(2)
column rank, XOR rows, and sequential counter. It checks the emitted XNF
clause-for-clause and, for a SAT transcript, checks every assignment and the
full tensor. [`test_225_union_subset_sat.py`](test_225_union_subset_sat.py)
exhausts all primary and auxiliary assignments of a small counter, proving
that its semantics are at-most rather than exactly-k. A positive control on
the five-door dictionary with limit 18 returned an independently replayed
rank-18 scheme in 12 ms. Limit 17 was solver-reported UNSAT in 13 ms; this
small result was not promoted into a global lower-bound claim.

Measured rank-17 searches were:

| dictionary / solver | encoding | budget | result |
|---|---|---:|---|
| maximin 32 / CryptoMiniSat | XOR + sequential `<=17`, default | 300 s | indeterminate, about 2.34M conflicts |
| maximin 32 / CryptoMiniSat | same, Gaussian matrices disabled | 300 s | indeterminate, 3,627,753 conflicts |
| maximin 32 / Z3 | native Bool-Xor + `PbLe(17)` | 60 s | timeout, 1,411,683 conflicts |
| maximin 32 / Z3 | native Bool-Xor + `PbEq(17)` | 60 s | timeout, 1,301,266 conflicts |
| all 4,096 / CryptoMiniSat | XOR + sequential `<=17`, default | 300 s | indeterminate, about 1.08M conflicts |

The Z3 equality arm is sound only because it first runs the pinned checked
`R >= 17` audit and therefore knows that a `rank <= 17` solution would use
exactly seventeen terms. `solve_225_union_z3.py --exact` refuses to run
without both the explicit `--lb17-certified` acknowledgement and a successful
`n225_verify_wang.py --audit-only` replay. The default Z3 and Tungsten arms
retain the unconditional `<=17` semantics.

The 30-second CryptoMiniSat calibration processed 191,240 conflicts with
false polarity, 217,089 with random polarity, 224,203 with forced Gaussian
elimination, and 397,829 with Gaussian matrices disabled. More conflicts did
not produce a model or a proof; these variants remain offline experiments.

## Why rank-18 odd-parent affine descent cannot close the gap

[`flipfleet_225_union_affine_descent.w`](flipfleet_225_union_affine_descent.w)
builds the exact zero-code generated by the same parents. The 4,101 serialized
inputs reduce to 3,750 distinct exact schemes, 3,749 nonzero generator rows,
and affine-code dimension 1,625 over 3,321 term coordinates. Tempered searches
covered 17,709,209 coefficient evaluations without improving rank 18/d84.

There is a stronger algebraic obstruction: every parent has even rank 18.
The symmetric difference of any odd number of them also has even cardinality.
It can therefore never land at rank 17. This family is useful for same-rank
diversity or a two-rank jump, but it is the wrong hot lane for the one-rank
225 gap unless an odd-rank exact parent or a non-affine term rewrite enters
the move.

For completeness, the offline tool audited every odd XOR of three distinct
generated schemes. If `A,B,C` each have 18 terms, then

```text
|A xor B xor C|
  = 54 - 2(|A intersect B| + |A intersect C| + |B intersect C|)
       + 4|A intersect B intersect C|.
```

A result at most 17 is even, hence at most 16. Rearranging gives

```text
sum(pair intersections) - 2*triple_intersection >= 19,
```

so at least one pair intersection is at least seven. This proves the
completeness of the scan's `overlap >= 7` filter: triples with no qualifying
pair are excluded analytically, and every possible low-weight triple has at
least one qualifying pair that enters the enumerator. Among 7,029,375 scheme
pairs, 218,571 passed. For each such pair the tool scanned every distinct
third scheme, making 819,204,108 completing-pair probes in 13.088 seconds in
a release/native replay. An unordered triple with multiple qualifying pairs
can therefore be probed more than once. Minimum rank remained 18. Five exact
anchor-pair scans also checked 35,128,130 generator pairs without a rank drop.
These counts are a reproducible finite audit, not an additional tensor-rank
lower bound; the already checked rank-17 lower bound independently excludes
the only possible even improvement, rank 16.

No SAT or affine lane from this audit is scheduled in the production pool.
The next exact fixed-dictionary attack now exists as
[`AFFINE_DECODER_225.md`](AFFINE_DECODER_225.md): a complete tensor-column
kernel decoder with sparse rank-band walks and randomized Lee--Brickell
information sets. Unlike the even-parent lane above, its 413-dimensional
basis contains 184 odd-weight rows, so rank 17 is reachable in principle.
The first 685.31 billion exact affine candidates still found no rank 17; that
bounded negative is not a dictionary proof. Adding genuinely new rank-one
terms outside the block-local dictionary remains the complementary path.
