# Contributing to Tungsten

### Reporting a Security Vulnerability

Please **do not use the issue tracker to report security vulnerabilities** as that would create the opportunity for possible exploits.

Instead, send an email to thecompanygardener@gmail.com.

## Using the issue tracker

### Reporting a Bug

The [issue tracker](https://github.com/tungsten-lang/tungsten/issues) is the best place to start. First, glance over the existing issues to see if any appear similar. search for similar existing issues, it may be `in-development`, `fixed`, or `needs-review`.

> If your bug report has security implications, please read the following section.

Over time, changes to the language will be introduced slowly, carefully. Please do not expect fast changes to the syntax and semantics.

### Reporting Other Issues

If you are having an issue with the website, or the mailing list, you can contact
the project lead to help resolve the problem.

The current project lead is:

* Erik Peterson (thecompanygardener@gmail.com)
